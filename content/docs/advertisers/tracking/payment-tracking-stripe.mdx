## Affitor Pay

Redirect affiliate-referred customers to Affitor's hosted checkout. Affitor handles payment collection and tracking automatically.

:::tip[Program ID]
When you copy code from your Affitor dashboard (**Settings â†’ Tracking**), your Program ID is pre-filled. If copying from these docs, replace `YOUR_PROGRAM_ID` with your ID from Settings.
:::

### Implementation

```javascript
// Check if customer was referred by an affiliate
if (window.affitor && window.affitor.partnerCode) {
  // Customer WAS referred - use Affitor checkout
  window.affitor.redirectToCheckout({
    affiliateProgramId: YOUR_PROGRAM_ID,   // REQUIRED: your Program ID
    amount: 299.99,                         // REQUIRED: purchase amount
    checkoutType: 'subscription',           // Optional: 'payment' or 'subscription'
    subscriptionInterval: 'monthly',        // Optional: 'monthly' or 'annual'
    productId: 'prod_ABC123',               // Optional: Stripe product ID
    priceId: 'price_ABC123'                 // Optional: Stripe price ID
  });
} else {
  // Customer was NOT referred - use your own checkout
  handleYourOwnCheckout({
    amount: 299.99,
    productId: 'prod_ABC123'
  });
}
```

:::caution[Conditional Checkout]
Only use Affitor Pay for customers who arrived via an affiliate link. Check `window.affitor.partnerCode` to determine if the customer was referred. Direct (organic) customers should use your normal checkout flow.
:::

### Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `affiliateProgramId` | number | Yes | Your Affitor Program ID |
| `amount` | number | Yes | Purchase amount (must match priceId value if provided) |
| `checkoutType` | string | No | `'payment'` or `'subscription'` (default: `'payment'`) |
| `subscriptionInterval` | string | No | `'monthly'` or `'annual'` (for subscriptions) |
| `productId` | string | No | Stripe product ID |
| `priceId` | string | No | Stripe price ID |
| `returnUrl` | string | No | Custom return URL after checkout |

### How It Works

1. Customer clicks buy â†’ `redirectToCheckout()` called
2. Affitor creates Stripe Checkout session with tracking metadata
3. Customer completes payment on Affitor-hosted page
4. Webhook fires â†’ conversion tracked â†’ commission calculated
5. You receive payment minus commission and platform fee

**Pros:** Zero Stripe configuration needed
**Cons:** By design, only referred customers use Affitor checkout â€” organic customers use your normal flow

### Option A: Client-Side (Quick Start)

Good for testing and simple setups. Reads tracking cookies on the client and passes them to your checkout endpoint.

#### One-Time Payment

```javascript
// Helper to read Affitor cookies
function getCookie(name) {
  const value = `; ${document.cookie}`;
  const parts = value.split(`; ${name}=`);
  if (parts.length === 2) return parts.pop().split(';').shift();
  return null;
}

const customerCode = getCookie('customer_code');

// Pass tracking data to your checkout endpoint
const response = await fetch('/api/create-checkout', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    priceId: 'price_xxx',
    mode: 'payment',
    // REQUIRED: tracking fields
    customer_code: customerCode,
    user_id: currentUser.id,             // REQUIRED: your internal user ID
    program_id: 'YOUR_PROGRAM_ID'        // REQUIRED: your Program ID
  })
});

const { url } = await response.json();
window.location.href = url;
```

### Option B: Server-Side (Recommended for Production)

Most enterprise SaaS use this approach. More secure â€” keeps tracking logic on your server.

#### One-Time Payment

```javascript
// In your checkout API endpoint (Node.js example)
app.post('/api/create-checkout', async (req, res) => {
  // Extract customer_code from cookies (sent in request headers)
  const cookies = req.headers.cookie || '';
  const customerCode = cookies.split('; ')
    .find(row => row.startsWith('customer_code='))
    ?.split('=')[1];

  // Get user from your authentication system
  const userId = req.user.id;

  const session = await stripe.checkout.sessions.create({
    line_items: [{ price: 'price_xxx', quantity: 1 }],
    mode: 'payment',
    success_url: 'https://yoursite.com/success',
    cancel_url: 'https://yoursite.com/cancel',

    // REQUIRED: All three tracking fields
    metadata: {
      customer_code: customerCode,
      user_id: userId,
      program_id: process.env.AFFITOR_PROGRAM_ID
    }
  });

  res.json({ url: session.url });
});
```

#### Subscription

```javascript
app.post('/api/create-checkout', async (req, res) => {
  const cookies = req.headers.cookie || '';
  const customerCode = cookies.split('; ')
    .find(row => row.startsWith('customer_code='))
    ?.split('=')[1];

  const userId = req.user.id;

  const session = await stripe.checkout.sessions.create({
    line_items: [{ price: 'price_xxx', quantity: 1 }],
    mode: 'subscription',
    success_url: 'https://yoursite.com/success',
    cancel_url: 'https://yoursite.com/cancel',

    // REQUIRED: Session metadata (first payment)
    metadata: {
      customer_code: customerCode,
      user_id: userId,
      program_id: process.env.AFFITOR_PROGRAM_ID
    },

    // REQUIRED: Subscription metadata (recurring payments)
    subscription_data: {
      metadata: {
        customer_code: customerCode,
        user_id: userId,
        program_id: process.env.AFFITOR_PROGRAM_ID
      }
    }
  });

  res.json({ url: session.url });
});
```

:::tip[Required Tracking Fields]
Include all three fields (`customer_code`, `user_id`, `program_id`) in Stripe metadata for reliable payment attribution. The backend uses multiple lookup methods to ensure tracking works even if cookies are blocked.
:::

### How It Works

1. Customer clicks affiliate link â†’ tracked with internal cookies
2. Customer signs up â†’ calls `trackLead({ user_id })` with your internal user ID
3. Customer clicks buy â†’ your checkout with `user_id` in metadata
4. Customer completes payment via your Stripe
5. Stripe webhook fires â†’ Affitor receives event
6. Affitor looks up customer-partner using `user_id` â†’ attributes to partner
7. Commission calculated â†’ Affitor invoices you weekly

**Pros:** Simple integration, use your existing checkout, no cookie reading required
**Cons:** Requires metadata setup, weekly invoice payment

## Metadata Reference

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `customer_code` | string | **Yes** | Customer identifier from Affitor cookie (set automatically when user clicks affiliate link) |
| `user_id` | string | **Yes** | Your internal user ID (must match the value from `trackLead()`) |
| `program_id` | string | **Yes** | Your Affitor program ID |

**All three fields are required for reliable payment attribution.**

:::caution[Required Metadata Fields]
All three metadata fields must be included:
- `customer_code` - Read from Affitor cookie (primary lookup method)
- `user_id` - Your internal user ID (fallback lookup + validation)
- `program_id` - Your Affitor program ID (program identification)

Without these fields, the purchase cannot be attributed to the affiliate partner.
:::

### Important Notes

- **Customer Code:** Set automatically by Affitor tracker when user clicks affiliate link. Read from cookie using the helper function.
- **User ID Must Match:** The `user_id` must exactly match the value you passed during `trackLead()`.
- **Program ID:** Find your program ID in the Affitor dashboard under Settings.
- **Hybrid Attribution:** Backend tries `customer_code` first (primary), falls back to `user_id` if cookies are blocked.

## Troubleshooting

### Conversion Not Tracked

**Check:**
- All three required fields included in Stripe metadata:
  - `customer_code` (from Affitor cookie)
  - `user_id` (your internal user ID)
  - `program_id` (your Affitor program ID)
- Cookie reading function working correctly
- Cookies not blocked by browser (check browser console)
- `user_id` matches the value used in `trackLead()` during signup
- Webhook endpoint configured correctly
- Webhook events selected (`checkout.session.completed`)
- User signed up via affiliate link (customer-partner record exists)

### Wrong Partner Attributed

**Check:**
- Customer clicked partner's link before signing up
- `trackLead()` was called during signup with correct `user_id`
- Customer-partner relationship exists in database
- `customer_code` in metadata matches the cookie value
- `user_id` in Stripe metadata exactly matches signup `user_id`

### Subscription Renewals Not Tracked

**Check:**
- `subscription_data.metadata` includes all three fields:
  - `customer_code`
  - `user_id`
  - `program_id`
- `invoice.payment_succeeded` webhook event selected
- Partner within commission duration window
- Original checkout has customer-partner relationship

### Why Multiple Lookup Steps?

| Scenario | Lookup Method | Example |
|----------|---------------|---------|
| Normal flow | `customer_code` (Step 1) | Customer has cookies intact |
| Cookies cleared | `email` (Step 2) | Customer cleared browser data |
| Returning customer | `stripe_customer_id` (Step 3) | Customer made previous purchase |
| Cross-device | `email_hash` (Step 3) | Customer clicked on mobile, bought on desktop |
| Custom tracking | `advertiser_user_id` (Step 4) | You provided custom user ID |

### What Affitor Receives from Stripe

When the webhook fires, Affitor extracts:

- Payment confirmation
- `program_id` from metadata
- `customer_code` from metadata (if available)
- `user_id` from metadata (if available)
- Customer email from Stripe
- `stripe_customer_id` from Stripe

### After Successful Lookup

Once a customer-partner relationship is found:

1. Verifies attribution is within 60-day window
2. Creates commission record
3. Updates `stripe_customer_id` if not already set (Step 2)

### Organic Sales

If no attribution is found after all 4 steps, the payment is considered an **organic sale** and is **NOT tracked** (discarded). This ensures only affiliate-referred sales generate commissions.
