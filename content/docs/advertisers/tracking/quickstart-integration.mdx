Get affiliate tracking live in three steps. Each step builds on the previous one: clicks → signups → sales.


### Install the Script

Add the Affitor script to every page of your site. It auto-detects affiliate links (`?aff=` parameter) and stores the `click_id` in a cookie for later attribution.

```html
<script
  src="https://api.affitor.com/js/affitor-tracker.js"
  data-affitor-program-id="YOUR_PROGRAM_ID"
  data-affitor-debug="false">
</script>
```

Replace `YOUR_PROGRAM_ID` with your program ID from the dashboard.

**What this does:**

- Detects `?aff=` in the URL
- Stores a `click_id` in a first-party cookie (60-day window)
- Fires a `pageview` event to Affitor — partners can see their click counts

**Verify:** Open your browser console. You should see:

```
[Affitor] Tracker loaded — program: YOUR_PROGRAM_ID
[Affitor] Attribution stored — click_id: cust_...
```


### Track Signups

After a user successfully registers, notify Affitor so the signup is linked to the referring affiliate. Choose the method that fits your stack:

### Option A: Browser-Side (Script)

Call `signup()` from the frontend after the user registers. Requires the Affitor script from Step 1.


  

```html
<script>
// Call this after your signup/registration succeeds
await window.affitor.signup('user_123', 'user@example.com');
</script>
```

  
  

```html
<script>
document.getElementById('signup-form').addEventListener('submit', async function(e) {
  e.preventDefault();

  var email = document.getElementById('email').value;

  // Run your own signup logic first, then on success:
  await window.affitor.signup(currentUser.id, email);
});
</script>
```

  


### Option B: Server-Side API

Send a `POST` request from your backend after registration completes. Recommended for backend-heavy integrations.


  

```bash
POST https://api.affitor.com/api/v1/track/lead
Authorization: Bearer YOUR_PROGRAM_API_KEY
Content-Type: application/json

{
  "click_id": "cust_42_1234567890",
  "customer_key": "user_123",
  "email": "user@example.com"
}
```

  
  

```js
await fetch('https://api.affitor.com/api/v1/track/lead', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.AFFITOR_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    click_id: req.cookies.affitor_click_id,
    customer_key: newUser.id,
    email: newUser.email,
  }),
});
```

  


:::note
`customer_key` is required for both options. It lets Affitor match the signup to a later payment even if the user checks out with a different browser or device.
:::

**What this does:**

- Validates the stored `affitor_click_id` cookie
- Creates a lead record linked to the referring partner
- Partners can now see: Clicks → **Signups**


### Track Sales

When a customer pays, Affitor needs to know. Choose the method that fits your checkout:

### Option A: Server-Side API

Send a `POST` request from your backend whenever a sale completes. Works with any payment provider or custom checkout.

**Endpoint:** `POST https://api.affitor.com/api/v1/track/sale`

**Request body:**

```json
{
  "transaction_id": "txn_abc123",
  "customer_key": "user_123",
  "click_id": "cust_42_123456",
  "amount_cents": 9999,
  "currency": "USD",
  "sale_type": "payment",
  "is_recurring": false,
  "subscription_id": "sub_xyz",
  "subscription_interval": "monthly",
  "product_id": "prod_456",
  "line_items": [
    { "name": "Pro Plan", "amount_cents": 9999, "quantity": 1 }
  ]
}
```

| Parameter | Required | Type | Description |
|-----------|----------|------|-------------|
| `transaction_id` | **Yes** | string | Unique ID for this transaction — used for deduplication |
| `amount_cents` | **Yes** | number | Positive integer, amount in cents (e.g. `$99.99` = `9999`) |
| `customer_key` | Conditional | string | Your internal customer ID. **At least one** of `customer_key` or `click_id` must be provided |
| `click_id` | Conditional | string | From `affitor_click_id` cookie. **At least one** of `customer_key` or `click_id` must be provided |
| `currency` | No | string | `USD`, `EUR`, or `VND`. Default: `USD` |
| `sale_type` | No | string | `payment` or `subscription`. Default: `payment` |
| `is_recurring` | No | boolean | `true` for recurring charges. Default: `false` |


  

```js
await fetch('https://api.affitor.com/api/v1/track/sale', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${process.env.AFFITOR_API_KEY}`,
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    transaction_id: order.id,
    customer_key: currentUser.id,
    click_id: req.cookies.affitor_click_id,
    amount_cents: order.totalCents,
    currency: 'USD',
    sale_type: 'payment',
  }),
});
```

  
  

```json
{
  "success": true,
  "sale_id": 42,
  "commission_id": 18,
  "message": "Sale tracked successfully"
}
```

  


:::note
`transaction_id` must be unique per sale — Affitor uses it for deduplication. If you send the same `transaction_id` twice, the second request will be rejected with a `409 Conflict`.
:::

### Option B: Stripe Checkout (Automatic)

If you use Stripe, pass Affitor metadata in your checkout session. Affitor's webhook listener handles the rest automatically — no extra API call needed.

```js
// Server-side — Node.js example
const session = await stripe.checkout.sessions.create({
  line_items: [{ price: 'price_xxx', quantity: 1 }],
  mode: 'payment',
  success_url: 'https://yoursite.com/success',
  cancel_url:  'https://yoursite.com/cancel',
  metadata: {
    affitor_click_id: 'cust_xyz789',
    affitor_customer_key: 'user_123',
    program_id:    'YOUR_PROGRAM_ID',
  },
});
```

**What this does:**

- Matches the payment to the correct partner via `click_id`
- Calculates commission based on your program tier
- Partners can now see: Clicks → Signups → **Sales**

## You're Done

Once all three steps fire successfully, your integration is complete. You can verify it end-to-end using the [Testing Integration](/advertisers/tracking/testing-integration/) guide.

| Step | Event | Dashboard view |
|------|-------|---------------|
| Script installed | `pageview` | Clicks |
| Signup tracked | `lead` | Signups |
| Payment tracked | `conversion` | Revenue |
