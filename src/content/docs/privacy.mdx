# Privacy & Data Protection

**Last Updated:** February 2, 2026  
**Compliance Standards:** GDPR, CCPA

---

## Overview

Affitor is committed to protecting the privacy and security of personal data. This document outlines our privacy practices, data handling policies, and compliance with GDPR (General Data Protection Regulation) and CCPA (California Consumer Privacy Act).

---

## What Data We Collect

### Event Tracking Data

When users visit advertiser websites through affiliate links, we collect:

- **Attribution Data:** Customer codes, partner codes, referral link information
- **Geographic Data:** Country, region, city (derived from IP address)
- **Browser Data:** User agent, screen resolution, viewport size, language, timezone
- **Session Data:** Page URL, page title, referrer URL, session ID

**What we DO NOT collect:**
- ❌ IP addresses (converted to geolocation immediately)
- ❌ Full email addresses (only hashed and masked versions)
- ❌ Customer names
- ❌ Phone numbers
- ❌ Payment method details
- ❌ Shipping addresses

### Customer Data (Lead Events)

When customers sign up or make purchases:

- **Email Hash:** SHA256 hash for matching/deduplication (one-way, irreversible)
- **Email Masked:** Partially masked email for display (e.g., `j***@gmail.com`)
- **Customer Code:** Unique identifier for tracking
- **User ID:** If customer is registered on advertiser's platform

### Commission & Payment Data

For affiliate commission calculations:

- **Purchase Amount:** Transaction value in USD/EUR/VND
- **Currency:** Transaction currency
- **Stripe Session ID:** Checkout session identifier
- **Commission Amount:** Calculated affiliate earnings
- **Payment Status:** Commission payment status

## Events Without Attribution

**Critical Privacy Rule:** We only store events that have valid affiliate attribution.

Events without valid attribution (missing customer_code or invalid referral link) are **rejected and not stored**. This ensures:

- ✅ Data minimization principle (GDPR Article 5)
- ✅ Purpose limitation - data collected only for affiliate tracking
- ✅ No collection of non-affiliate user behavior

### Attribution Validation

All events must pass validation before storage:

**Click Events:**
- ✅ Referral link must exist and be active
- ✅ Referral link must belong to an active partner program

**Lead/Conversion Events:**
- ✅ Customer code must exist in database
- ✅ Customer must have associated referral link
- ✅ Referral link must be active

## Email Handling Standard

### How We Protect Email Addresses

1. **Hashing (SHA256):**
   - One-way cryptographic hash
   - Used for email matching and deduplication
   - Cannot be reversed to retrieve original email
   - Example: `john@example.com` → `973dfe463ec857...` (64 characters)

2. **Masking:**
   - Partial obfuscation for display
   - Partners see masked version only
   - Example: `john@example.com` → `j***@example.com`

3. **No Plaintext Storage:**
   - Original email addresses are never stored
   - Received from Stripe webhooks, immediately hashed/masked
   - Plaintext discarded after processing

### Email Search & Matching

Partners can search customers by:
- ✅ Customer code (full or partial match)
- ✅ Masked email (partial match)
- ✅ Exact email (hashed and compared)

Partners **cannot:**
- ❌ Browse all customer emails
- ❌ Export plaintext email lists
- ❌ Access emails from other partners' customers

## IP Address & Geolocation

### Privacy-First Approach

**IP addresses are never stored.** Instead, we immediately convert them to geographic data:

```typescript
// IP address received
const ipAddress = "8.8.8.8";

// Immediately converted to geolocation
const geo = await ipToGeo(ipAddress);
// { country: "US", region: "California", city: "Mountain View" }

// Only geolocation stored, IP discarded
await createTrackingEvent({
  geo_country: geo.country,
  geo_region: geo.region,
  geo_city: geo.city
  // ip_address: NOT STORED
});
```

### Geolocation Service

- **Provider:** ip-api.com (free tier: 45 requests/minute)
- **Caching:** 24-hour in-memory cache to reduce API calls
- **Fallback:** Returns null values for localhost/invalid IPs
- **Upgrade Option:** MaxMind GeoIP2 for production at scale

## Webhook Payload Storage

### Safe Metadata Only

When processing Stripe webhooks, we store **only essential metadata**:

**Stored:**
- ✅ Event type (e.g., `checkout.session.completed`)
- ✅ Event ID (Stripe event identifier)
- ✅ Timestamp
- ✅ Object type and ID (e.g., `checkout_session`, `cs_123...`)
- ✅ Routing decision (which service handled it)

**NOT Stored:**
- ❌ Full webhook payload
- ❌ Customer email/name/phone from payload
- ❌ Payment method details
- ❌ Billing/shipping addresses

For debugging, use **Stripe Dashboard** to view full webhook details.

## Logging & Monitoring

### PII-Safe Logging

Application logs automatically redact sensitive fields:

```typescript
// Unsafe logging (OLD)
console.log({ email: "user@example.com", name: "John" });
// Output: { email: "user@example.com", name: "John" } ❌

// Safe logging (NEW)
safeLogger.info({ email: "user@example.com", name: "John" });
// Output: { email: "[REDACTED]", name: "[REDACTED]" } ✅
```

**Auto-Redacted Fields:**
- `email`, `customer_email`, `customerEmail`
- `name`, `customer_name`, `customerName`
- `phone`, `customer_phone`, `customerPhone`
- `ip_address`, `ipAddress`
- `address`, `street`, `postal_code`, `ssn`

## Data Ownership Model

### Three-Party Relationship

1. **Advertiser (Data Controller):**
   - Owns customer relationship
   - Controls customer data via Stripe Connect account
   - Responsible for customer consent and privacy notices

2. **Affitor (Data Processor):**
   - Processes attribution and commission data on behalf of advertiser
   - Stores only hashed/masked customer identifiers
   - Cannot access full customer PII

3. **Affiliate Partner (Sub-Processor):**
   - Receives commission for referred customers
   - Sees only masked customer data
   - Cannot access customer PII

### Access to Customer Data

| Data Type | Advertiser | Affitor | Partner |
|-----------|-----------|---------|---------|
| Full Email | ✅ (via Stripe) | ❌ | ❌ |
| Email Hash | ✅ | ✅ | ❌ |
| Email Masked | ✅ | ✅ | ✅ |
| Customer Name | ✅ (via Stripe) | ❌ | ❌ |
| Purchase Amount | ✅ | ✅ | ✅ |
| Commission Amount | ✅ | ✅ | ✅ |

## Frequently Asked Questions

### Why don't you store email addresses?

Email addresses are personally identifiable information (PII). By storing only hashed and masked versions, we:
- Reduce privacy risk
- Minimize data breach impact
- Comply with data minimization principles

### Can partners see customer emails?

No. Partners see only masked emails (e.g., `j***@gmail.com`). Full customer details are available only to advertisers via their Stripe Connect account.

### How do you match customers without storing emails?

We use SHA256 hashes. When a customer checks out, we hash their email and match it against existing hashes in our database.

### What if I need to contact a customer?

Advertisers have access to full customer details via Stripe Connect. Partners should contact the advertiser to relay messages to customers.

### Do you comply with GDPR?

Yes. We follow GDPR principles including data minimization, purpose limitation, and user rights. We have a Data Protection Officer and provide Data Processing Agreements.

### How do I delete my customer data?

Contact the advertiser whose program you signed up for. As the data controller, they will submit a deletion request to us.
