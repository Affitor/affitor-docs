## Enable Test Mode

### npm SDK

Pass `debug: true` when loading the tracker:

```ts
import { loadAffitor } from '@affitor/tracker';

const affitor = await loadAffitor('YOUR_PROGRAM_ID', { debug: true });
```

Or with the React provider:

```tsx
<AffitorProvider programId="YOUR_PROGRAM_ID" debug>
  {children}
</AffitorProvider>
```

### Script Tag

Add `data-affitor-debug="true"` to your tracker script:

```html
<script
  src="https://api.affitor.com/js/affitor-tracker.js"
  data-affitor-program-id="YOUR_PROGRAM_ID"
  data-affitor-debug="true">
</script>
```

**Test mode:**
- Logs all tracking events to browser console
- Sends test events only — no real records created in database
- Test events visible in dashboard for verification

## Test Checklist

### 1. Pageview Tracking (Click)

**Steps:**
1. Open browser Developer Tools (F12) → Console
2. Visit your site (no `?aff=` needed — the test event fires automatically in debug mode)
3. Look for Affitor debug messages

**Expected console output:**

```
[Affitor] Affitor tracker v2 initialized: {debug: true}
[Affitor] Using program ID: YOUR_PROGRAM_ID
[Affitor] Verifying cookie functionality...
[Affitor] Cookie set: {name: "affitor_test_cookie", value: "test_...", expires: ...}
[Affitor] Cookie verification PASSED
[Affitor] trackTest() called true
[Affitor] Sending test click event: {affiliate_url: ..., page_url: "...", ...}
```

**If cookies are blocked**, you'll see:

```
[Affitor] Cookie verification FAILED - cookies may be blocked
[Affitor]   Possible causes: browser privacy settings, ITP, extensions
```

:::caution[Do not use ?aff= in test mode]
Do **not** add `?aff=TEST123` to the URL when testing. Click tracking (`trackClick()`) is not guarded by debug mode — it will create a **real** click record and set real cookies. The `trackTest()` event fires automatically and is sufficient for dashboard verification.
:::

**Dashboard check:**
- Go to Affitor dashboard → **Settings → Pageview Tracker**
- Click **Test Click Tracking**
- Pageview tracker should show a blue checkmark

### 2. Lead Tracking (Signup)

**Steps:**
1. With cookies set (from step 1), complete your signup flow
2. Ensure you pass `customerKey` in your `signup()` call (required for payment attribution)
3. Check console for lead event

**Expected console output:**

```
[Affitor] trackTest() called true
[Affitor] Sending test lead event: {click_id: ..., customer_key: "your_customer_key", session_id: "...", ...}
[Affitor] Test lead event sent successfully {...}
```

**Note:** In debug mode, `signup()` sends a test event only — no actual lead record is created. If `customerKey` is missing, you'll see:

```
[Affitor] signup() requires customerKey as first argument
```

**Dashboard check:**
- Go to Affitor dashboard → **Settings → Referrals Tracker**
- Click **Test Lead Tracking**
- Referrals tracker should show a blue checkmark

### 3. Payment Tracking (Sale)

**Steps:**
1. Use Stripe test mode (test card: `4242 4242 4242 4242`)
2. Complete a test purchase
3. Check Stripe Dashboard → Webhooks for delivery status

**Verify metadata (Stripe Dashboard):**
- Go to Payments → find test payment
- Click to view details
- Check metadata contains:
  - `customer_code: customer_code_from_cookie`
  - `user_id: your_user_id` (must match the value from `trackLead()`)
  - `program_id: YOUR_PROGRAM_ID`

**Dashboard check:**
- Go to Affitor dashboard → **Settings → Payments Tracker**
- Stripe Connect Integration should show "Connected"
- Payments tracker should show a blue checkmark

## Test API Endpoint

Programmatically verify integration:

```http
POST https://api.affitor.com/api/tracking/test
Content-Type: application/json

{
  "program_id": "YOUR_PROGRAM_ID",
  "step_id": "pageview"
}
```

**Response:**

```json
{
  "success": true,
  "message": "Test event received",
  "testEvent": {
    "id": "evt_123",
    "event_type": "test",
    "timestamp": "2026-01-21T12:00:00Z"
  }
}
```

## Common Test Scenarios

### Prerequisites

Before running these scenarios, you need a partner with a referral link:

1. Go to Affitor dashboard → **Partners**
2. Approve for your test partner
3. In the Partner Dashboard -> **Referral Links** -> **Create Link**
4. Copy the referral link — you'll use this in the scenarios below

For Scenario 3 (Partner Override), you need **two** partners with separate referral links.

### Scenario 1: Full Flow Test

These scenarios test **real tracking** (debug mode off) with actual affiliate links:

1. Open incognito window (fresh cookies)
2. Visit the partner's referral link (e.g. `yoursite.com?aff=abc123`)
3. Sign up with test email
4. Purchase with Stripe test card
5. Check dashboard for: 1 click → 1 signup → 1 conversion

### Scenario 2: Returning Visitor

1. Visit the partner's referral link (e.g. `yoursite.com?aff=abc123`)
2. Leave site
3. Return directly to `yoursite.com` (no ref)
4. Purchase

**Expected:** Attributed to the partner (customer-partner relationship persists via cookie)

### Scenario 3: Partner Override

1. Visit Partner A's referral link (e.g. `yoursite.com?aff=abc123`)
2. Later visit Partner B's referral link (e.g. `yoursite.com?aff=xyz789`)
3. Purchase

**Expected:** Attributed to Partner B (last click wins - relationship updated)

### Scenario 4: Expired Attribution

1. Visit the partner's referral link
2. Wait 61+ days
3. Purchase

**Expected:** No attribution (outside 60-day window)

## Troubleshooting Test Issues

### Console Shows No Affitor Messages

**Check:**
- `data-affitor-debug="true"` is set
- Script URL is correct
- No ad blockers/privacy extensions

### Cookies Not Setting

**Check:**
- URL has `?aff=` parameter
- Site uses HTTPS
- No Content Security Policy blocking cookies
- No browser extensions blocking cookies

### Webhook Not Firing

**Check:**
- Stripe Connect integration is connected in Settings → Payments Tracker
- Correct webhook events selected (`checkout.session.completed`, `invoice.payment_succeeded`)
- All three metadata fields included in Stripe checkout (`customer_code`, `user_id`, `program_id`)

### Test Event Not in Dashboard

**Check:**
- Program ID is correct
- Backend API is reachable
- Check Network tab for failed requests

## Going Live

Once all tests pass:

1. **Remove test mode:**
   - npm SDK: Remove `debug` prop or set `debug={false}`
   - Script tag: Remove `data-affitor-debug` attribute or set to `"false"`

2. Switch Stripe to live mode (if using test mode)

3. Verify one real transaction before announcing to partners

4. Monitor dashboard for first few days
